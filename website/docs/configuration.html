CONFIGURATION

Master the Loopwork configuration system

Configuration File

Loopwork uses a loopwork.config.ts (or .js) file for configuration. The configuration follows a composable pattern similar to Next.js.

typescript 
import { compose, defineConfig } from 'loopwork'
import { withJSONBackend } from 'loopwork/backends'

export default compose(
  withJSONBackend({
    tasksFile: '.specs/tasks/tasks.json'
  }),
)(defineConfig({
  cli: 'claude',
  maxIterations: 50,
}))

The Compose Pattern

The compose() function allows you to chain multiple plugins together. Each plugin wraps and enhances the configuration.

Plugins are applied in order - the first plugin in the compose chain wraps the base config, and subsequent plugins wrap around that.

Core Configuration Options

Option Type Default Description
cli string 'claude' CLI command to use (claude, opencode, gemini)
maxIterations number 10 Maximum number of tasks to process
timeout number 600 Timeout per task in seconds
namespace string 'default' Namespace for task isolation
dryRun boolean false Preview tasks without executing

CLI Configuration

Configure multiple AI models with automatic failover:

typescript 
import { compose, defineConfig, withCli, ModelPresets } from 'loopwork'

export default compose(
  withCli({
    models: [
      ModelPresets.claudeSonnet({ timeout: 300 }),
      ModelPresets.geminiFlash({ timeout: 300 }),
    ],
    fallbackModels: [
      ModelPresets.claudeOpus({ timeout: 600 }),
    ],
  }),
)(defineConfig({}))

Environment Variables

Create a .env file in your project root:

env 
# AI API Keys
ANTHROPIC_API_KEY=your_key_here
OPENAI_API_KEY=your_key_here
GOOGLE_API_KEY=your_key_here

# Telegram Notifications
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

# Discord Notifications
DISCORD_WEBHOOK_URL=your_webhook_url

# Asana Integration
ASANA_ACCESS_TOKEN=your_token
ASANA_PROJECT_ID=your_project_id

# GitHub Backend
GITHUB_TOKEN=your_token

Advanced Configuration Example

typescript 
import { 
  compose, 
  defineConfig, 
  withCli, 
  withGitHubBackend,
  withTelegram,
  withCostTracking,
  withGitAutoCommit,
  ModelPresets 
} from 'loopwork'

export default compose(
  // Use GitHub Issues as task source
  withGitHubBackend({
    repo: 'myorg/myrepo',
    label: 'loopwork-task',
  }),
  
  // Configure AI models with failover
  withCli({
    models: [
      ModelPresets.claudeSonnet({ timeout: 300 }),
      ModelPresets.geminiFlash({ timeout: 300 }),
    ],
    fallbackModels: [
      ModelPresets.claudeOpus({ timeout: 600 }),
    ],
  }),
  
  // Enable notifications
  withTelegram({
    botToken: process.env.TELEGRAM_BOT_TOKEN,
    chatId: process.env.TELEGRAM_CHAT_ID,
    notifyOnComplete: true,
    notifyOnFail: true,
  }),
  
  // Track costs
  withCostTracking({
    enabled: true,
    dailyBudget: 10.00,
  }),
  
  // Auto-commit changes
  withGitAutoCommit({
    enabled: true,
    addAll: true,
  }),
)(defineConfig({
  maxIterations: 100,
  timeout: 900,
}))
