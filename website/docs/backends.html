TASK BACKENDS

Flexible task sources for your automation

Backend Overview

Loopwork supports multiple task backends. A backend is responsible for storing and retrieving tasks, tracking their status, and managing dependencies.

JSON Backend

The JSON backend stores tasks in a local JSON file with accompanying Markdown description files.

typescript 
import { withJSONBackend } from 'loopwork/backends'

withJSONBackend({
  tasksFile: '.specs/tasks/tasks.json',
})

JSON Task Structure

json 
{
  "tasks": [
    {
      "id": "TASK-001",
      "status": "pending",
      "priority": "high",
      "dependencies": []
    },
    {
      "id": "TASK-002",
      "status": "pending",
      "priority": "medium",
      "dependencies": ["TASK-001"]
    }
  ]
}

Task Description Format

Each task has a corresponding Markdown file with the detailed description:

markdown 
# TASK-001: Create Hello World function

## Goal
Create a simple hello world function in TypeScript.

## Requirements
- Function should accept a name parameter
- Return "Hello, {name}!"
- Include TypeScript types
- Add basic tests

## Success Criteria
- Function compiles without errors
- Tests pass
- Code follows project style

GitHub Backend

Use GitHub Issues as your task source. Perfect for teams already using GitHub for project management.

typescript 
import { withGitHubBackend } from 'loopwork/backends'

withGitHubBackend({
  repo: 'myorg/myrepo',
  label: 'loopwork-task',
  state: 'open',
})

GitHub Issue Labels

Label Purpose
loopwork-task Marks issue as a Loopwork task
priority:high Sets high priority
priority:medium Sets medium priority
priority:low Sets low priority
feature:* Feature area for filtering

Notion Backend

Sync tasks with a Notion database for teams using Notion as their knowledge base.

typescript 
import { withNotionBackend } from 'loopwork/backends'

withNotionBackend({
  databaseId: 'your-database-id',
  statusProperty: 'Status',
  pendingStatus: 'Not started',
  completedStatus: 'Done',
})

Custom Backend

Build your own backend by implementing the TaskBackend interface:

typescript 
import { TaskBackend, Task } from 'loopwork/contracts'

class MyCustomBackend implements TaskBackend {
  async findNextTask(): Promise<Task | null> {
    // Find and return the next pending task
  }

  async getTask(id: string): Promise<Task | null> {
    // Get a specific task by ID
  }

  async markCompleted(id: string): Promise<void> {
    // Mark a task as completed
  }

  async markFailed(id: string, error: string): Promise<void> {
    // Mark a task as failed with error
  }

  async getDependencies(id: string): Promise<string[]> {
    // Get task dependencies
  }

  async areDependenciesMet(id: string): Promise<boolean> {
    // Check if all dependencies are completed
  }
}
The backend system is designed to be extensible. You can integrate with Jira, Linear, ClickUp, or any other task management system.
